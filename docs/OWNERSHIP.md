# Module Ownership — CareGo Healthcare Platform

> **Purpose**: Defines which developer owns which directories and files.  
> Ownership means: **only the owner should edit files in that directory.** If the other developer needs a change, they request it via GitHub Issue or PR comment.

---

## Developer Roles

| Role | Alias | Responsibilities |
|------|-------|-----------------|
| **Dev A** | Backend Developer | Encore.ts services, database, API design, migrations |
| **Dev B** | Frontend/Mobile Developer | Flutter app, React web frontends, UI/UX |

---

## Directory Ownership Map

### 👤 Dev A — Backend & Infrastructure

| Directory / File | Notes |
|-----------------|-------|
| `backend/` (entire tree) | All Encore.ts services |
| `backend/db/migrations/` | ⚠️ **Exclusive** — only Dev A creates migrations |

| `backend/auth/` | Auth service |
| `backend/user/` | User service |
| `backend/ambulance/` | Ambulance service |
| `backend/admin/` | Admin service |
| `backend/caregiver/` | Caregiver service |
| `backend/rental/` | Rental service |
| `backend/app/` | App version service |
| `docs/API_CONTRACT.md` | Dev A writes, Dev B reads |
| `docs/MIGRATION_LOG.md` | Dev A maintains |

### 👤 Dev B — Flutter & Web Frontends

| Directory / File | Notes |
|-----------------|-------|

## Ownership Violation Protocol

If you need to edit a file in another developer's directory:

1. **Open a GitHub Issue** explaining what change is needed and why.
2. **Tag the owner** for review.
3. **The owner makes the change** or approves a PR from you.
4. **Exception**: Typo fixes and comment updates can be done via PR without prior coordination.
